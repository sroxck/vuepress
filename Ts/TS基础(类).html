# TS基础-类

TS类大体和ES6的类相同,使用 extends 继承,使用super调用父类,多了一些特性

类的基本操作,继承和调用父类

class Lady {
    content="你好"
    sayHello(){
        return this.content
    }
}
// 继承父类并重写方法
class Goddess extends Lady{
    // 重写父类方法
    sayHello(){
        return super.sayHello() + "JOJO"
    }
    sayHi(){
        return this.content + "hi!"
    }
}
console.log(new Goddess().sayHello());
console.log(new Goddess().sayHi());

# 类的访问类型

# public

公共的,在类的内部和外部都可以调用,

# protected

受保护的,只能在类的内部使用,也可以在继承中的类中使用

# private

私有的,只能在类的内部使用,继承的类的内部也不可用

TIP

不写修饰符默认是public类型

不同修饰符的访问权限

class Demo{
    protected uname = "sroxck1"  // 受保护的
    private userName = "sroxck2"  // 私有的
}
// let demoExample1 = new Demo().uname // 报错 不可访问 受保护的
// let demoExample2 = new Demo().userName // 报错 不可访问 私有的
class DemoSon extends Demo{
    log(){
        console.log(this.uname) // 成立 protected 可以在继承的类中访问
        // console.log(this.username) // 报错 private 只能在本类中使用
    }
}

# 类的构造函数继承

  1. 子类继承父类的时候,子类如果要写构造方法,必须手动调用super()去执行父类的构造方法,
  2. super()此语句必须放在第一行(先有父,再有子)
class DemoExtends {
    constructor(public name:string){}
}
class DemoExtendsSon extends DemoExtends {
    constructor(public age:number){
        super("sroxck:jdd") // 如果子类要写构造方法,必须调用父类的构造方法
        // 父类没有构造方法也必须调用,因为会有一个默认的空的构造函数
    }
}
console.log(new DemoExtendsSon(22).age,new DemoExtendsSon(22).name);

# 类的getter和setter

设置get和set后,可以直接通过对象.属性的方式去.get的方法名去获取值而不用调用方法
set 直接通过set方法名 赋值即可,可以把get和set方法的名字看做是一个成员变量,访问和修改时,会执行对应的包装函数

class Shower{
    constructor(private showerTime:number){}
    get shower(){
        return this.showerTime - 10
    }
    set shower(num:number){
        this.showerTime = num+10
    }
}

const xizao = new Shower(30)
xizao.shower = 40
console.log(xizao.shower);

# 类的只读属性

class ReadonlyClass{
    readonly _name:string
    constructor(name:number){
        this._name=name as unknown as string
    }

console.log(new ReadonlyClass(22)._name);
// 给类的成员变量添加readonly修饰符即可只读

# 使用接口来描述类的方法和属性

类使用接口需要使用implements关键字

interface discriptionClass {
    currentTime:Date;
    setTime(d: Date):Date
}
// 使用implements实现接口,类中必须包含接口中声明的变量和方法
class TimeTool implements discriptionClass{
    result:any;
    currentTime:any
   
    setTime(date:any){
        return date
    }
}
let ss = new TimeTool()
console.log(ss.setTime(new Date()));
console.log(ss.currentTime);
// 接口只能检查类的公共部分,不会检查公共和私有两部分

# 抽象类

使用abstract声明抽象类,使用extends继承抽象类,抽象类无法单独使用,必须被继承

abstract class DemoDemo{
    // # names:string='33
    names:string="33"
    abstract logerr():void
    abstract sf:string
    aha(){
        console.log(222);
    }
}
class demodemo extends DemoDemo{
    sf='33'
    logerr(){
        console.log("33");
    }
}
new demodemo().logerr()
console.log(new demodemo().names);
new demodemo().aha()

抽象类格式:

  1. 格式: 在类的声明前加上abstract 即可声明抽象类
  2. 抽象方法 方法名前加abstract 并且不用写方法体,也可以定义抽象属性,但是不能定义静态抽象
  3. 抽象方法中可以定义成员变量,也可以定义普通方法和静态方法
  4. 继承抽象类,需要实现里面全部的抽象方法和抽象属性

注意

在类中使用private的私有方法,可以通过as any来访问,因为背后编译出来的是自执行函数的作用域私有化,是可以被重构的,而使用#号声明私有成员属性,是不会被访问到的,因为背后用的是wakeMap封装,使用#的方式,需要改变lib的值为ES6,target也需要改为ES6并且添加DOM,因为ts在target为ES6的时候是不允许访问dom的
